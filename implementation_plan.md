# 抽屉 · Drawer — 重设计方案 v2

## 一、产品重定位

**核心洞察**：Drawer 不是 AI 思维完善工具，而是一个**思维速记本 + 自动归档师**。

> 用户真正的使用流程：
> **触发（看到一件事）→ 快速倾倒（跟 AI 聊）→ 生成点子卡（结构化总结）→ 带着卡片去深聊/写/做**

AI 的角色从"苏格拉底式挑战者"变为**"速记员"**——目的是在用户还热着的时候，尽快把灵感挖完、抓牢。

## 三、重设计方向

### 3.1 视觉重心：聊天 → 点子卡

| 之前 | 之后 |
|------|------|
| 聊天占满主区域 | **点子卡** 占满主区域 |
| 图谱 300px 右栏 | 图谱合并入点子卡或折叠 |
| 聊天是主角 | 聊天是工具（字面"抽屉"——拉开倒入，关上归档） |

### 3.2 布局方案（方案 A）

**桌面端（≥768px）：**
```
[点子列表 240px] | [点子卡主区域 flex] | [聊天抽屉 从右/底滑出]
```

**手机端（<768px，Mobile-first）：**
```
[点子列表 全屏] → 点击进入 → [点子卡 全屏] → 点击按钮 → [聊天从底部滑上来]
```

### 3.3 点子卡（核心新功能）

每个"点子"打开后，主区域展示一张**动态点子卡**，包含：

| 区块 | 内容 | 生成方式 |
|------|------|---------|
| 核心想法 | 1-2 句话，这个点子的本质是什么 | AI 自动，对话后实时更新 |
| 关键发散 | 3-5 个分支方向（bullet） | AI 自动提炼 |
| 未解决的张力 | 矛盾或卡点是什么 | AI 自动 |
| 可能的下一步 | 最小化行动建议 | AI 自动 |
| 对话时间线 | 钉入的关键节点，竖排时序 | 用户手动钉入 |

卡片提供**一键复制**功能，可直接粘贴给 Claude 或作为写作大纲。

### 3.4 AI Prompt 重写

```
你是"抽屉"——用户的思维速记员。

目标：在用户还热着的时候，帮他/她把灵感挖完、抓牢。
不是挑战，不是教学，是帮你"把脑子里的东西全倒出来"。

做法：
- 追问用细节：「这个X具体是什么意思？」
- 帮他发现自己没意识到说出来的东西：「你刚才同时提到了A和B，它们有关系吗？」
- 拓宽覆盖：「除了这个方向，还有什么你没说到的？」
- 够了就停：如果核心想法已经清晰，说一句「好，我帮你整理成卡片了」

简洁、口语、有温度。不列清单，不说教，每次只问一个问题。
```

### 3.5 保留 / 修改 / 去掉

| 功能 | 决定 |
|------|------|
| 点子列表（左栏） | ✅ 保留 |
| 聊天对话 | ✅ 保留，改为底部抽屉 |
| D3 图谱 | 🔄 降级为点子卡内的"时间线"，图谱视图可选 |
| Todo 方形节点 | ❌ 移除（不符合核心需求） |
| confetti 撒花 | ❌ 移除（配套 Todo 功能） |
| 钉入时间线（圆形节点） | ✅ 保留，但视觉改为卡片内时间线 |
| 30分钟定时提醒 | ✅ 保留 |
| `generateHook` | ✅ 保留 |
| 点子卡生成 | 🆕 新功能 |
| 点子卡一键复制 | 🆕 新功能 |
| PWA 支持 | 🆕 新增（`manifest.json` + `service-worker.js`） |

## 四、开发阶段规划

### Phase 1：修 Bug + Prompt 重写（低风险，快）
1. **修时间错位 Bug**：`pinToTimeline` 接收调用时的 `chatHistory` 快照
2. **重写 AI System Prompt**：速记员角色
3. **DESIGN_JOURNAL.md 更新**：补充本次产品重定位决策

### Phase 2：点子卡（核心新功能）
4. **新增点子卡区块**：替换聊天为主区域
5. **卡片自动生成逻辑**：对话达到一定轮数后，AI 自动提炼卡片内容
6. **卡片手动触发**：用户也可点击"整理成卡片"
7. **一键复制**

### Phase 3：聊天变抽屉
8. **布局重构**：点子卡为主区域，聊天从底部/侧面滑出
9. **Mobile-first 响应式**：手机布局优先
10. **D3 图谱改为时间线视图**：嵌入点子卡内

### Phase 4：PWA
11. **`manifest.json`**：图标、名字、主题色
12. **`service-worker.js`**：离线缓存
13. **部署到 Vercel/Netlify**（可选）

## 五、Verification Plan

### 每个 Phase 的浏览器手动验证
- 在 Chrome 中直接打开 `抽屉 (10).html`（或新文件名）进行测试
- 无自动化测试基础设施，全部为手动验证

### Phase 1 验证
1. 打开页面，和 AI 聊几轮
2. 在第 2 轮消息下点击"钉入时间线"→ 确认提炼内容是第 2 轮附近的内容，而非最新对话
3. 确认 AI 回复风格变化：更像速记员，追问而非挑战

### Phase 2 验证
4. 和 AI 聊 4+ 轮后，确认点子卡自动生成
5. 点击"整理成卡片"手动触发，确认卡片内容合理
6. 点击"一键复制"，粘贴到记事本确认格式正确

### Phase 3 验证
7. 打开页面，确认主区域是点子卡，聊天默认不可见
8. 点击聊天按钮，确认聊天面板从底部滑出
9. 在 Chrome DevTools 中切换至手机尺寸（375px），验证手机布局正确
